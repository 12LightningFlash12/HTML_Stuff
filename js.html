Basics:

First let's understand what JavaScript does within webpages; JavaScript conrols the website's behavior. This covers things like buttons, DOM or Document Object Model Events, changing HTML content depending on some form of input, etc. Basically, if it makes the webpage behave in a certain way, that is done using JavaScript.


Explanations



1. Adding JavaScript in webpages

To add JavaScript, the <script> </script> tag must be used, but there is two ways to do this. First,the JavaScript can go inside the script tag (fig. 1.1). Or, an outside JavaScript file can be linked using the href attribute (fig. 1.2).


2. JavaScript DOM

2.1 What DOM Is

DOM Events, or Document Object Model, allows event-driven programming languages, such as JavaScript, to register variuos events. (fig. 2.1) shows the DOM Tree of Objects. With the DOM Tree, JavaScript has enough power to create a dynamic webpage. With this power, JavaScript can do the following:

  • Change the HTML elements
  • Change the HTML attributes
  • Change the CSS styles
  • Remove HTML elements\attributes
  • Add HTML elements\attributrs
  • React to HTML events
  • Create new HTML events

2.2 DOM Methods

In DOM, all HTML elements are defined as objects; methods and properties can be defined for each object. A method is an action that can be done, and a property is a value that can be set. Examples(fig 2.2.1):

  • Properties: changing content
  • Methods: adding\deleting content

2.3 DOM Elements

Using JavaScript, you can find, access, and minipulate any HTML element. You can find elements by:

  • Id (fig. 2.3.1)
  • Tag name (fig. 2.3.2)
  • Class name (fig. 2.3.3)
  • CSS selectors (fig. 2.3.4)
  • HTML object collections (fig. 2.3.5)

2.4 DOM Events

JavaScript can react to certain events. When the event is triggered, the corresponding JavaScript code is executed. Examples of such events:

  • Mouse click (fig. 2.4.1)
  • Webpage loading (fig. 2.4.2)
  • Image loading (fig. 2.4.3)
  • Mouse moving over an element (fig. 2.4.4)
  • Changing an input field (fig. 2.4.5)
  • Submitting an HTML form (fig. 2.4.6)
  • Keystrokes (fig. 2.4.7)
